# Workflow Authoring Rules

- Workflows are SOPs -- write them like you're briefing a capable person, not a robot
- Each workflow must include: objective, required inputs, tools to use, expected output, edge cases
- When you discover better methods or constraints (rate limits, timing quirks), update the workflow immediately
- Don't create new workflows without asking RA -- these are instructions that need to be preserved
- Don't overwrite existing workflows without asking -- append or propose changes instead
- Archive outdated workflows to `workflows/archive/` -- never delete
- Keep workflow names descriptive: what it does, not what it's called
