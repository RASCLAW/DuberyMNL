# Skill — Pre-Compact Guide

## When to use
Invoke this before typing /compact when you want to guide what gets preserved
in the summary. Takes 30 seconds and prevents important context from being lost.

## What it does
1. Asks RA what to focus on
2. Saves key items to memory
3. Gives the all-clear to compact

---

## Execution

Ask RA this question:

"Before we compact — what's the most important thing happening right now that
the summary needs to get right? (e.g., 'we're testing the gatekeeper on 019',
'we just fixed X and are about to run Y', 'the issue is Z')"

Wait for RA's answer.

Then:
1. Save any new facts, decisions, or state to the appropriate memory file
   (feedback, project, or reference type)
2. If there's active in-progress work, add a brief note to PROJECT_LOG.md
   under the current session entry (or create one if none exists today)
3. Confirm: "Saved. You can type /compact now — the summary will prioritize [what RA said]."

## Rules
- Do not summarize the whole session here -- that's /compact's job
- Only save what RA explicitly calls out as important
- Keep it fast -- this should take under 2 minutes
